# 🐠 Tropical Fish · מערכת התנדבות למשמרות סופ"ש

אפליקציית web בעברית לניהול התנדבות למשמרות שישי-שבת, לחברת בריכות דגים ומים (~30 עובדים).

## 🌐 הקישור לאפליקציה

**https://yardensate.github.io/tropical-fish-shifts/**

שלחו את הקישור לעובדים (למשל בקבוצת WhatsApp). בטלפון אפשר "להוסיף למסך הבית" ואז זה מתנהג כמו אפליקציה.

## 🚀 התחלה — 3 צעדים

1. **פתיחה ראשונה:** מי שפותח ראשון את הקישור רואה מסך "הגדרה ראשונית" — שם יוצרים את חשבון **המנהל הראשון** (שם, טלפון וקוד מנהל בן 4 ספרות). עשו זאת מיד אחרי הפרסום.
2. **הוספת עובדים:** בלשונית **עובדים** לוחצים "הוספת עובד" וממלאים שם פרטי, שם משפחה וטלפון. זה הכול.
3. **שיתוף:** שלחו לעובדים את הקישור. הם נכנסים עם השם והטלפון שלהם — בלי סיסמה.

## 👤 איך זה עובד לעובד

- נכנסים פעם אחת (שם + טלפון). האפליקציה זוכרת אתכם במכשיר.
- רואים את לוח החודש, עם ניווט חופשי **שנה שלמה אחורה וקדימה** (אפשר למשל להירשם כבר עכשיו לחגי תשרי). ימי חול אפורים; **שישי, שבת וחגים מודגשים** — חגים מסומנים בזהב עם שם החג.
- **התנדבות:** לוחצים על שישי/שבת/חג עתידי ובוחרים **"אני רוצה לעבוד"** או **"אני לא רוצה לעבוד"**. סימון "רוצה" נשלח לאישור מנהל. בלוח רואים את המצב בצבע: 🟢 מאושר · 🟠 ממתין · 🔴 לא רוצה.
- **דיווח שעות:** לוחצים על כל יום בשבוע — עבר, היום או קדימה (אפשר לדווח מראש) — מזינים משעה ועד שעה (למשל 08:00–18:00), והמערכת מחשבת מיד את הפירוק לשעות רגילות / 125% / 150%. מתחת ללוח מופיע סיכום חודשי אישי.

## 🧮 חוקי חישוב השעות

| סוג יום | שעות בסיס (100%) | אחר כך |
|---|---|---|
| יום רגיל (א׳–ה׳) | 9 שעות | שעתיים ב־125%, ומעבר לכך 150% |
| שישי / ערב חג | 7 שעות | שעתיים ב־125%, ומעבר לכך 150% |
| שבת / חג ושבתון (כמו פסח) | 8 שעות | שעתיים ב־125%, ומעבר לכך 150% |

- אם חג נופל על יום חול — היום מקבל את חוקי החג. חג שנופל על שבת נשאר בחוקי שבת (לעולם לא "מוריד" את הסיווג).
- משמרת שמסתיימת אחרי חצות (למשל 22:00–06:00) מחושבת לפי היום שבו התחילה.
- **החגים כבר בפנים:** כל חגי ישראל (ערבי חג, ימי חג ושבתון, יום העצמאות) לשנים 2026–2028 טעונים מראש מלוח Hebcal. מנהל יכול להוסיף/להסיר ימים בעצמו.

## 🛡️ איך זה עובד למנהל

מנהל נכנס עם שם + טלפון **+ קוד בן 4 ספרות** (נשמר במכשיר אחרי הפעם הראשונה). למנהל 4 לשוניות:

| לשונית | מה יש שם |
|---|---|
| **לוח משמרות** | כל הסימונים של כולם על הלוח, עם מונים לכל יום. לחיצה על יום פותחת פירוט + אישור/דחייה. **כאן גם מוסיפים יום התנדבות:** לוחצים על כל יום בלוח, נותנים שם ובוחרים סוג (רגיל 9 ש׳ / כמו ערב חג 7 ש׳ / כמו חג 8 ש׳). אפשר גם להסיר יום מיוחד (כולל חג מהלוח) |
| **בקשות** | כל הבקשות שממתינות לאישור, מקובצות לפי תאריך, עם כפתורי אשר/דחה. מספר הבקשות מופיע על הלשונית |
| **שעות** | דוח שעות חודשי לכל עובד: סה"כ, 100%, 125%, 150%. לחיצה על עובד פותחת פירוט יומי. כפתור **"דוח PDF"** פותח דוח מעוצב להדפסה — טבלת סיכום לכל העובדים + פירוט יומי לכל עובד — ובחלון שנפתח בוחרים "שמירה כ-PDF" (Save as PDF) |
| **עובדים** | הוספה, עריכה ומחיקה של עובדים; הפיכת עובד למנהל (כולל קביעת קוד) |
| **המשמרות שלי** | הלוח האישי של המנהל, לסימון משמרות ודיווח שעות לעצמו |

הגנות מובנות: אי אפשר למחוק את עצמך, אי אפשר להסיר ניהול מעצמך, וחייב להישאר לפחות מנהל אחד.

## 🔒 אבטחה — בשפה פשוטה

- זהו כלי פנימי לצוות קטן: הכניסה היא שם+טלפון בלי סיסמה, ולכן **אל תפרסמו את הקישור מחוץ לחברה**.
- קודי המנהלים נשמרים מוצפנים (hash) — לא ניתן לקרוא אותם מהמסד.
- הרשאות המסד פתוחות לטבלאות `fish_` בלבד (פשרה מודעת לטובת פשטות בכלי פנימי ל-30 עובדים). אם בעתיד תרצו הקשחה — אפשר להעביר את הגישה דרך Supabase Edge Functions.

## 🛠️ למפתחים — עבודה מקומית ועדכונים

```bash
npm install        # התקנה חד-פעמית
npm run dev        # שרת פיתוח: http://localhost:5173
npm test           # בדיקות יחידה לעזרי התאריכים
npm run build      # בניית גרסת פרודקשן לתיקיית dist
```

**עדכון האתר החי** אחרי שינוי בקוד:

```bash
npm run build
cd dist
git init && git checkout -b gh-pages && git add -A && git commit -m "Deploy"
git push -f https://github.com/yardensate/tropical-fish-shifts.git gh-pages
cd ..
```

(ואת שינויי הקוד עצמם דוחפים ל-`main` כרגיל.)

## 🗄️ מבנה הנתונים (Supabase)

- `fish_employees` — עובד: שם פרטי, שם משפחה, טלפון (ייחודי), האם מנהל, hash של קוד המנהל.
- `fish_shift_requests` — סימון: עובד + תאריך (ייחודי יחד), העדפה (`want`/`dont_want`), סטטוס (`pending`/`approved`/`rejected`/`noted`), מי החליט ומתי.
- `fish_time_entries` — דיווח שעות: עובד + תאריך (ייחודי יחד), שעת התחלה ושעת סיום. הפירוק ל־100/125/150 מחושב באפליקציה לפי סוג היום.
- `fish_special_days` — ימים מיוחדים: תאריך (ייחודי), שם, סיווג שכר (`regular`/`friday`/`saturday`), מקור (`holiday` = נטען מלוח החגים, `manual` = נוסף ע"י מנהל). חגי 2026–2028 כבר טעונים; להרחבה לשנים הבאות אפשר להוסיף ידנית מהאפליקציה או לבקש מ-Claude לטעון מ-Hebcal.
